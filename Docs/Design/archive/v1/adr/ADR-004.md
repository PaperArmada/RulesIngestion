# ADR-004: Pairing Edges — Delta/Base and Exception/Base (Instrumented; Not Yet Proven)

**Status:** Accepted  
**Date:** 2026-02

## Context

Dependency-oriented pairing edges (delta→base, exception→base) can add candidates to the ranked list when a delta or exception unit is retrieved, to pull in its base. This may improve recall for rule-modifier and exception queries.

## Decision

Pairing edges are **instrumented** in the retrieval path (dual-list + pairing config) but are **not** claimed as a proven improvement. Every run with pairing must emit per-query (or per-run) counters: pairing triggers fired, candidates added by pairing, gold added by pairing (when gold set known), whether added candidates entered top-10, dedupe interactions. Evaluation of pairing benefit is based on this instrumentation; headline metrics alone are insufficient.

## Consequences

- Config `phb_hybrid_dual_list_fusion_plus_pairing.yaml` is labeled experimental.
- Run output must include pairing instrumentation so that benefit can be evaluated even when headline MRR/Recall do not move.
- No T1 regressions allowed; pairing must not degrade T1 vs dual-list fusion.
