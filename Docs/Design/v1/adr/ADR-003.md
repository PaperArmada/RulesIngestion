# ADR-003: Clause Families Are Retrieval Projections (Not Admissible)

**Status:** Accepted  
**Date:** 2026-02

## Context

Clause families group an anchor EvidenceUnit with nearby units (by structural_path and order) to improve recall for compositional queries. This creates a second “list” (Index_F) that can be fused with the canonical EvidenceUnit list (Index_U).

## Decision

Clause families are **retrieval-only projections**. They are not admissible evidence. When both an EvidenceUnit and its family anchor (or a family member) appear in the candidate set, dedupe priority is: **EvidenceUnit preferred over family anchor**. Families are used only to improve ranking and recall; the units returned as evidence remain EvidenceUnits.

## Consequences

- Retrieval Lab may index and retrieve family anchors/members for comparison and recall metrics, but the canonical admissible layer for citation is EvidenceUnits only.
- Schema and docs clearly label ClauseFamily as a projection with anchor_unit_id, members, params.
