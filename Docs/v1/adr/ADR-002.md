# ADR-002: Dual-List Fusion Is Production Default

**Status:** Accepted  
**Date:** 2026-02

## Context

Hybrid retrieval (dense + sparse RRF) can be run with a single combined list or with two lists (Index_U = canonical EvidenceUnits, Index_F = clause-family projection) fused with configurable k. Comparisons showed dual-list fusion improves recall without sacrificing T1 performance when tuned.

## Decision

For PHB (and where applicable), **dual-list fusion** is the **production default** retrieval config. The baseline hybrid (single list, no dual-list) remains available for comparison and regression testing. Default config: `phb_hybrid_dual_list_fusion.yaml`.

## Consequences

- New experiments and production retrieval use dual-list fusion unless explicitly comparing to baseline.
- T1 regression count vs baseline must remain 0 unless waived by ADR.
- Baseline runs are preserved and documented in baseline_manifest.md.
